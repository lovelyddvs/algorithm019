# 动态规划
特点1：重复子问题：因为存在大量重复子问题，所以需要记录之前的结果
特点2：最优子结构：不同问题规模之间的关系
特点3：无后效性，只记录之前的结果，而不关心这个结果是怎么来的，一般而已，状态定义越仔细，就能清楚后效行。
思路方向1：自顶向下 -- 递归 + 记忆化：少数问题这样子，刚开始学习时这样做，逐渐过渡到自底向上。
思路方向2：自底向上 -- 递推求解：绝大多数问题需要这样做，需要多练习。
